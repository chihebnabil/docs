## ğŸ¯ Usage Guide

### Testing Individual Files (Free & No Auth Required)

Perfect for beginners! Test how your components will be converted without using credits or signing up.

<CodeGroup>
```bash Basic Conversion
# Convert a single component file
next-lovable convert MyComponent.tsx
```

```bash Preview Changes
# See what changes will be made without modifying the file
next-lovable convert MyComponent.tsx --dry-run --show-diff
```

```bash Save to Different Location
# Convert and save to a new location
next-lovable convert MyComponent.tsx --output ./converted/MyComponent.tsx
```
</CodeGroup>

### Full Project Migration

<Warning>
  Full project migration requires a nextlovable.com account and consumes credits. Always test individual files first!
</Warning>

<Steps>
  <Step title="Run a Dry Run First">
    ```bash
    next-lovable migrate ./my-react-app --dry-run
    ```
    This shows you exactly what will be changed without making any modifications.
  </Step>
  
  <Step title="Execute the Migration">
    ```bash
    next-lovable migrate ./my-react-app ./next-app --yes --install
    ```
    This creates a new Next.js project and installs dependencies.
  </Step>
</Steps>

## ğŸ”„ What Gets Converted

### React Router â†’ Next.js Router

<CodeGroup>
```tsx Before (React Router)
import { Link, useNavigate, useLocation } from 'react-router-dom';

export const Navigation = () => {
  const navigate = useNavigate();
  const location = useLocation();

  return (
    <nav>
      <Link to="/dashboard">Dashboard</Link>
      <Link to={`/profile/${userId}`}>Profile</Link>
      <button onClick={() => navigate('/settings')}>Settings</button>
      <p>Current: {location.pathname}</p>
    </nav>
  );
};
```

```tsx After (Next.js)
'use client';

import Link from 'next/link';
import { useRouter, usePathname } from 'next/navigation';

export const Navigation = () => {
  const router = useRouter();
  const pathname = usePathname();

  return (
    <nav>
      <Link href="/dashboard">Dashboard</Link>
      <Link href={`/profile/${userId}`}>Profile</Link>
      <button onClick={() => router.push('/settings')}>Settings</button>
      <p>Current: {pathname}</p>
    </nav>
  );
};
```
</CodeGroup>

### React Helmet â†’ Next.js Head

<CodeGroup>

## ğŸ›ï¸ Available Transformations

<AccordionGroup>
  <Accordion title="router - React Router Conversion">
    - `useNavigate()` â†’ `useRouter()` from `next/navigation`
    - `<Link to="/path">` â†’ `<Link href="/path">` from `next/link`
    - `useLocation()` â†’ `usePathname()` + `useSearchParams()`
    - `useParams()` â†’ Next.js `useParams()`
    - Template literals in paths are preserved
  </Accordion>

  <Accordion title="helmet - React Helmet Conversion">
    - `<Helmet>` â†’ `<Head>` from `next/head`
    - Meta tags and title preservation
    - SEO attributes maintained
  </Accordion>

  <Accordion title="client - Client Component Detection">
    - Automatic "use client" directive addition
    - Detects React hooks (useState, useEffect, etc.)
    - Identifies event handlers (onClick, onChange, etc.)
    - Recognizes browser APIs (window, document, localStorage)
  </Accordion>

  <Accordion title="context - Context Provider Migration">
    - Converts React Context providers
    - Maintains context functionality in Next.js structure
  </Accordion>
</AccordionGroup>

## ğŸ› Limitations & Best Practices

### Known Limitations
- Complex custom route configurations may need manual adjustments
- Dynamic route parameters might require review
- Some third-party integrations may need updates
- Complex state management setups might need fine-tuning

### Best Practices
1. **Always test individual files first** using the free `convert` command
2. **Run `--dry-run`** before any actual migration
3. **Keep backups** of your original project
4. **Review generated files** before deployment
5. **Test thoroughly** after migration
